The Forge VM is the core of virtualized protection. Forge generates a fresh interpreter for every build, with randomized opcodes, dispatch, and handler ordering.

## Runtime layout

```text
ForgeVM
├─ Registers
├─ Constant Pool
├─ Call Stack
├─ Closure Store
├─ Prototype Store
├─ Opcode Metadata
├─ Dispatcher
└─ Runtime Context
```

## Prototype format

```text
Prototype
├─ Instruction Stream
├─ Constants
├─ Upvalues
├─ Nested Prototypes
└─ Metadata
```

## Instruction descriptor

## Instruction mutation

Each opcode has multiple semantically equivalent handler implementations. The compiler picks one randomly per build, so:

- Two builds of the same source produce different VMs
- Static signature matching against the dispatcher fails
- Handler fingerprints can't be reused across builds
