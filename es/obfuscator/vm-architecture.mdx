La VM de Forge es el núcleo de la protección virtualizada. Forge genera un intérprete nuevo para cada compilación, con opcodes, despacho y orden de manejadores aleatorizados.

## Diseño en tiempo de ejecución

```text
ForgeVM
├─ Registers
├─ Constant Pool
├─ Call Stack
├─ Closure Store
├─ Prototype Store
├─ Opcode Metadata
├─ Dispatcher
└─ Runtime Context
```

## Formato del prototipo

```text
Prototype
├─ Instruction Stream
├─ Constants
├─ Upvalues
├─ Nested Prototypes
└─ Metadata
```

## Descriptor de instrucción

```text
Instruction
├─ Opcode
├─ Operand A
├─ Operand B
├─ Operand C
└─ Flags
```

## Mutación de instrucciones

Cada opcode tiene múltiples implementaciones de manejador semánticamente equivalentes. El compilador elige una al azar por compilación, de modo que:

- Dos compilaciones del mismo código fuente producen VMs diferentes
- La coincidencia de firmas estáticas contra el despachador falla
- Las huellas de los manejadores no pueden reutilizarse entre compilaciones
